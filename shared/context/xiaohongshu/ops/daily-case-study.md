# Daily XHS Case Study (Daily Ritual)

## Goal
Every day, learn from **one real high-performing XHS creator/post** in our niche (tennis fashion + lookbook/editorial), extract repeatable patterns, and turn it into **our own executable playbook**.

## Output (what we produce daily)
A single “Memory Entry” using the template:
- `shared/context/xiaohongshu/templates/memory-entry.md`

Stored under:
- `shared/context/xiaohongshu/case-studies/YYYY-MM-DD.md`

## Daily workflow (15–25 min)

### 0) Pick 1 case (5 min)
Choose one of:
- A top creator in niche (from `research.md` benchmarks)
- A “god post” (high save-rate / share-rate) found via keyword search

Selection criteria:
- Save-rate is strong OR saves > likes
- Format matches our positioning (lookbook/editorial, outfit formulas)

### 1) Capture evidence (3–5 min)
Record:
- Post link / id / screenshots (if needed)
- Basic metrics: likes / saves / comments / shares (if visible)
- The search keyword used + rank position (if via search)

### 2) Analyze with a fixed rubric (7–10 min)
Use this rubric:
- **Hook**: first line + cover promise
- **Cover**: composition, typography, “what’s the takeaway”
- **Content structure**: formula steps, list structure, length
- **Visual system**: palette, lighting, styling, props
- **Tags/keywords**: title words + hashtags matrix
- **Engagement ops**: pinned comment, Q&A pattern, CTA

### 3) Extract 3 takeaways + 1 experiment (3–5 min)
Write:
- 3 repeatable rules we can copy (not vibes)
- 1 experiment we run in the next 72h (post idea + angle)

### 4) Save to memory (2 min)
Create:
- `shared/context/xiaohongshu/case-studies/YYYY-MM-DD.md`

and add 1-line pointer into:
- `shared/context/xiaohongshu/README.md` change log (optional)

## Roles (recommended)
- **Thinker**: selects case + data notes + rubric analysis
- **Creator**: turns takeaways into template/cover + 1 post draft
- **Gatekeeper**: checks claims, security/privacy, and makes sure we captured evidence & next steps
- **Admin**: closes the loop: file saved + owners + due date, and posts daily summary in Slack

## Notes
- Phase 1: read-only (search/feeds/detail). No auto-publishing until MCP security + cookie health-check is in place.
