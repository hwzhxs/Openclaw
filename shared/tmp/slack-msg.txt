**AGENTS.md — 写行为协议，维度：团队级 / 操作性 / 跨 agent 可见**

写入标准：所有 agent 都需要遵守的规则才进这里。
颗粒度：具体到可执行，模糊规则没价值。

写入内容（按维度）：
- 协作协议：谁做什么（Creator 写代码，不是 Thinker）
- 工具约束：用什么工具做什么事（Slack 发消息用 slack-send.ps1，不用 raw IWR）
- 任务流程：任务如何流转（Thinker spec → Creator build → Gatekeeper QA）
- 禁止行为：全团队禁止项（不改 openclaw.json，不 DM Xiaosong）
- 路由规则：什么请求谁接（channel 收到 @all 的分工）
- 资源索引：共享路径、脚本位置、队列目录

不写入：个人人格、单个 agent 的价值观、模糊的"尽量做"建议

---

**SOUL.md — 写身份认同，维度：个人级 / 行为驱动 / 影响判断和语气**

写入标准：只有这个 agent 的特质，别人无需知道。
颗粒度：方向性而非操作性，但 "What I'm Not" 段要用硬规则语气（Never / Must not）。

写入内容（按维度）：
- 角色定位：我是什么（一句话身份）
- 价值观：我相信什么（驱动我做决定的原则）
- 工作方式：我怎么工作（决策优先级、习惯）
- 边界定义：我不做什么 — 这段要硬，不能模糊（Never / Must not）
- 团队习惯：我如何与队友互动（不是规则，是内化的行为模式）
- Growing Edge：我在哪里成长（帮 agent 自我校正）

不写入：操作流程、工具命令、团队协议

---

**一句话区别：**
AGENTS.md = 团队宪法（每条都是规则，可验证）
SOUL.md = 个人性格（每条都是驱动力，影响判断）

例：
"Creator 负责所有代码写入" → AGENTS.md
"我相信能跑的原型比完美的计划更有价值" → SOUL.md
