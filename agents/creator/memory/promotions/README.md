# memory/promotions/

## Purpose
Promoted memory items awaiting human review before they are merged into `MEMORY.md`.
Generated by `memory/scripts/consolidate.ps1`.

## Structure
```
promotions/
  YYYY-MM-DD--promoted.md   ← items ready for review on that date
```

## Rules
1. **Human approval required** — Xiaosong or Admin must approve before merging
2. **Do NOT auto-merge** — agents must not copy these directly to MEMORY.md
3. **One writer** — only the consolidation script writes here
4. **Append-only** — never overwrite existing promotion files; run script again to add more

## Approval Workflow
1. Reviewer reads `YYYY-MM-DD--promoted.md`
2. Removes or edits any items that shouldn't be promoted
3. Manually appends approved items to `MEMORY.md`
4. Archives or deletes the promotion file

## Conflict Handling
Items marked `⚠️ CONFLICT:` are **excluded** from promotions and require manual resolution.
See the corresponding `memory/reports/YYYY-MM-DD--consolidation.md` for details.
