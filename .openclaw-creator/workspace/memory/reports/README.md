# memory/reports/

## Purpose
Consolidation reports generated by `memory/scripts/consolidate.ps1`. Each report is a **read-only audit log** of a consolidation run.

## Structure
```
reports/
  YYYY-MM-DD--consolidation.md   ← one report per consolidation run
```

## Report Contents
Each report includes:
- **Run metadata** — date/time, candidate files processed
- **Promoted items** — new, non-conflicting items sent to `promotions/`
- **Conflicts** — items flagged `⚠️ CONFLICT:` for manual resolution
- **Skipped items** — already present in MEMORY.md
- **Summary stats** — X candidates, Y promoted, Z conflicts, W skipped

## Rules
1. **Never edit reports** — they are immutable audit records
2. **Review before acting** — check reports before approving promotions
3. **Conflicts require manual resolution** — never auto-resolve

## Workflow
```
candidates/ → consolidate.ps1 → reports/ (audit) + promotions/ (pending)
                                                          ↓
                                              Human review + approval
                                                          ↓
                                                    MEMORY.md update
```
