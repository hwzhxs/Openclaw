### Key Findings & Actions

#### 1. Mention+Webhook Gap (FIXED)
- **Problem:** Agents using `message(action=send)` with `<@U...>` mentions don't fire webhooks, so mentioned agents never wake up.
- **Root cause:** `message(action=send)` bypasses `slack-notify.ps1` which handles webhook firing.
- **Fix:** Hardcoded rule in ALL 4 agents' AGENTS.md: `message(action=send)` must NEVER contain `<@U...>`. All @mentions must go through `slack-notify.ps1`.
- **Additional fix:** Admin was excluded from `slack-notify.ps1` webhook map (line 32 comment "never webhook yourself"). Fixed: added Admin to map. Creator also added `-SenderUserId` param for self-skip logic.

#### 2. Watchdog.ps1 Corruption (FIXED)
- **Problem:** Creator's Module 12 edit corrupted watchdog.ps1 — the `catch` block at line 1011 had the entire original script (~900 lines) pasted into it as a string. File went from ~1010 valid lines to 1910 lines. Watchdog was broken for ~2 hours with no monitoring.
- **Fix:** I (Thinker) removed the corrupted lines 1011-1900, replaced with proper `} catch { Write-Host "Module 12 error: $_" }`. File went from 1910 to 1022 lines. Syntax check passed, run test passed.
- **Irony:** The oversight system itself went down due to an uncaught edit error — exactly the failure pattern we were discussing (Sam's #4).

#### 3. Auto-Detection + Auto-Fix Architecture (DESIGNED, NOT YET BUILT)
- **Current:** watchdog detects → alerts human/agent → waits
- **Desired:** watchdog detects → auto-fixes → verifies → only escalates to Xiaosong if fix failed
- **Who does what:**
  - Auto-detection: watchdog.ps1 (PowerShell scheduled task, every 2 min). No agent needed.
  - Auto-fix: mostly watchdog.ps1 directly (webhook fires, Slack API posts, file cleanup). Only "retry your own work" delegates back to agent via webhook.
  - Escalation: watchdog posts to Slack mentioning Xiaosong. Only when auto-fix fails twice or needs human judgment.
- **Auto-fixable directly by watchdog:** fire missing webhooks, restart gateways, delete .bak files, post reminders
- **Auto-fixable via agent delegation:** retry failed edits, nudge stuck agents, re-fire unanswered mentions
- **Escalate to Xiaosong only:** auto-fix failed twice, multiple agents down, judgment-required violations

#### 4. Tool Failure Visibility Gap (IDENTIFIED)
- **Problem:** When an agent's tool call fails, nothing captures it externally. Agent sees error in session but team doesn't know.
- **Current:** Module 9 (error detection) pattern `:warning:.*fail` should catch it, alerts Admin + offending agent. Gap: doesn't alert Xiaosong.
- **Xiaosong's directive:** "After detect violations, someone should auto fix it. I want auto detection and auto fix. If it really cannot be solved, then mention me."
- **Recommendation:** Add escalation path — 2+ errors in 30 min without fix → escalate to Xiaosong.
