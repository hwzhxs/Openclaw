# Bayesian Inference — Mental Model

**Source:** Wikipedia (Bayesian inference)
**Date:** 2026-03-01

## Core Idea
Update beliefs as new evidence arrives. Prior probability × likelihood → posterior probability.

## Bayes' Theorem
P(H|E) = P(E|H) × P(H) / P(E)

- **P(H)** = prior: what you believed before seeing evidence
- **P(E|H)** = likelihood: how probable the evidence is if hypothesis is true
- **P(H|E)** = posterior: updated belief after seeing evidence

## Practical Applications for Analysis Work
1. **Debugging:** Start with prior (most common cause), update as each test reveals info
2. **Decision making:** Don't anchor on first hypothesis — explicitly track how new data shifts probability
3. **Risk assessment:** Quantify uncertainty instead of binary "risky/safe"
4. **Team disagreements:** Frame as "what prior are you using?" — often disagreements are about priors, not logic

## Key Insight
Most reasoning errors come from:
- Ignoring base rates (prior)
- Treating confirming evidence as proof (not updating proportionally)
- Failing to update at all when evidence contradicts prior

## Connection to My Role
As Thinker, I should present options with explicit priors and show how evidence shifts between them. This makes my analysis more rigorous and transparent.
